# 2장. 손해 진전 속도 — 링크와 ATA 인자

이 장은 흐름의 **두 번째 단계**, 삼각형 위에서 손해의 진전을 진단하는
자리입니다. 진단 단계는 두 잣대로 나뉩니다 — 이 장의 ATA 인자(곱셈),
그리고 3장의 강도(덧셈).

```{mermaid}
flowchart LR
  A["경험 데이터"] --> B["Triangle<br/>삼각형 (1장)"]
  B --> C["link 진단<br/>ATA 인자 (이 장)"]
  C --> D["예측"]
  D --> E["불확실성"]
  E --> F["regime · 백테스트"]
  style C fill:#ffe08a,stroke:#d39e00,stroke-width:2px
```

1장에서 만든 직각삼각형은 오른쪽 아래가 비어 있었습니다. 최근에 인수된
코호트일수록 아직 경과 기간이 짧아 손해가 다 떠오르지 않았기
때문입니다. 담당자가 실제로 던지는 질문은 이것입니다.

> "이 코호트는 지금 손해가 이만큼인데, 앞으로 얼마나 더 자랄까?"

이 질문에 답하려면 먼저 **한 경과에서 다음 경과로 손해가 몇 배가
되는지**를 알아야 합니다. 그 배율이 이 장의 주인공인 **ATA 인자**입니다.

## 2.1 두 경과를 잇는 링크

같은 코호트를 경과 1에서 경과 2로, 다시 경과 3으로 따라가면 누적
손해가 점점 커집니다. 이때 **이웃한 두 경과 칸의 쌍**을 하나의
**링크**(link)로 봅니다. 링크 하나는 "이 코호트가 경과 1에서 경과 2로
갈 때 누적 손해가 얼마에서 얼마가 되었는가"를 담습니다.

그 변화를 한 숫자로 요약한 것이 **ATA 인자**(age-to-age factor, 한
경과에서 다음 경과로 누적 손해가 몇 배가 되는가)입니다.

$$
\text{ATA 인자} = \frac{\text{다음 경과의 누적 손해}}{\text{이번 경과의 누적 손해}}
$$

값이 1.5라면 한 경과를 지나는 동안 누적 손해가 1.5배, 즉 50% 더
자랐다는 뜻입니다. 1에 가까우면 거의 자라지 않은 것입니다.

## 2.2 손으로 구해 보는 ATA 인자

작은 삼각형으로 직접 계산해 보겠습니다. 코호트 셋(A·B·C)의 누적
손해입니다. 물음표는 아직 관측되지 않은 칸입니다.

```{list-table}
:header-rows: 1
:widths: 16 20 20 20

* - 코호트
  - 경과 1
  - 경과 2
  - 경과 3
* - A
  - 100
  - 150
  - 180
* - B
  - 120
  - 180
  - ?
* - C
  - 110
  - ?
  - ?
```

코호트 하나만 보면 우연이 섞이므로, **같은 링크에 있는 코호트들을 합쳐**
배율을 구합니다. 분자끼리 더하고 분모끼리 더한 뒤 나누는 것입니다(이를
**거래량 가중**(volume-weighted)이라 합니다).

**경과 1 -> 2** — 경과 1과 경과 2가 모두 관측된 코호트는 A·B입니다.

$$
f_1 = \frac{150 + 180}{100 + 120} = \frac{330}{220} = 1.5
$$

**경과 2 -> 3** — 경과 3까지 간 코호트는 A뿐입니다.

$$
f_2 = \frac{180}{150} = 1.2
$$

즉 이 포트폴리오에서는 경과 1을 지나며 손해가 1.5배, 경과 2를 지나며
1.2배 자랍니다. 곱셈 두 번이 전부입니다.

## 2.3 패키지로 확인하기

같은 삼각형을 `Triangle`에 넣고 ATA 인자를 뽑으면 손으로 구한 값과
같은지 확인할 수 있습니다.

```python
import polars as pl
import lossratio as lr

# 손계산용 예제 삼각형 (누적 손해)
toy = pl.DataFrame({
    "uy_m":    ["2024-01-01", "2024-01-01", "2024-01-01",
                "2024-02-01", "2024-02-01", "2024-03-01"],
    "cy_m":    ["2024-01-01", "2024-02-01", "2024-03-01",
                "2024-02-01", "2024-03-01", "2024-03-01"],
    "loss":    [100, 150, 180, 120, 180, 110],
    "premium": [1000, 1000, 1000, 1000, 1000, 1000],
}).with_columns(pl.col(["uy_m", "cy_m"]).str.to_date())

tri = lr.Triangle(toy, cohort="uy_m", calendar="cy_m",
                  loss="loss", premium="premium", cell_type="cumulative")

tri.link(target="loss").ata().df.select(["dev", "f", "n_cohorts"])
#> shape: (2, 3)
#> ┌─────┬─────┬───────────┐
#> │ dev ┆ f   ┆ n_cohorts │
#> │ --- ┆ --- ┆ ---       │
#> │ i64 ┆ f64 ┆ i64       │
#> ╞═════╪═════╪═══════════╡
#> │ 1   ┆ 1.5 ┆ 2         │
#> │ 2   ┆ 1.2 ┆ 1         │
#> └─────┴─────┴───────────┘
```

`f` 열이 손으로 구한 1.5·1.2와 정확히 같습니다. `n_cohorts`는 그 링크를
계산하는 데 쓰인 코호트 수입니다(경과 1->2는 2개, 경과 2->3은 1개).

```{admonition} 점추정은 손으로, 불확실성은 함수로
:class: note

이 예제 삼각형에서는 인자 값(점추정)이 산수만으로 깔끔하게
떨어집니다. 하지만 "이 1.5라는 값을 얼마나 믿을 수 있는가"를 측정하는
표준오차·변동계수 같은 불확실성 지표는 코호트 간 흩어짐을 회귀로
추정해야 하고, 코호트가 한둘뿐인 이 예제로는 측정할 수 없습니다. 이런
부분은 손으로 따라가지 않고 패키지가 계산하도록 맡깁니다(자세히는
5장에서). 이 튜토리얼의 손계산은 *담당자가 직접 따라갈 수 있는 산수*
까지만 하고, 회귀가 끼는 순간 함수에 넘깁니다.
```

빈 칸(B의 경과 3, C의 경과 2·3)을 실제로 채우는 일은 이 인자들을
곱해 나가면 됩니다. 그 과정은 4장에서 이 삼각형을 그대로 이어받아
손으로 마무리합니다.

## 2.4 실제 데이터에서 ATA 인자 읽기

내장 데이터의 SUR(수술) 담보로 같은 일을 해 봅니다. 경과별 ATA 인자를
나란히 그리면 손해 진전 속도가 경과에 따라 어떻게 변하는지 한눈에
들어옵니다.

```{eval-rst}
.. plot::
   :context:
   :nofigs:
   :include-source: false

   import polars as pl
   import lossratio as lr

   df = lr.load_experience().filter(pl.col("coverage") == "SUR")
   tri = lr.Triangle(df, groups="coverage", grain="Q")
   link = tri.link(target="loss", exposure="premium")

.. plot::
   :context: close-figs
   :caption: SUR 담보의 경과별 ATA 인자(분기 단위). 초기에는 손해가 빠르게 자라다 경과가 쌓일수록 1에 가까워진다.

   link.plot(model="ata", kind="summary")
```

초기 경과의 인자는 3.5로 큽니다. 갓 인수된 계약은 누적 손해가 작은
상태라, 한 기간만 지나도 몇 배로 불어나기 때문입니다. 경과가 쌓일수록
인자는 1.7, 1.4, 1.3처럼 점점 1에 가까워집니다. **인자가 1에 가깝다는
것은 손해가 더는 거의 자라지 않는다**는 뜻 — 곧 그 코호트의 손해가 다
떠올라 안정되어 간다는 신호입니다.

## 2.5 ATA 인자는 언제 믿을 만해지나

인자의 *크기*가 1에 가까워지는 것과는 별개로, 더 중요한 물음이
있습니다 — **같은 경과에서 코호트마다 인자가 얼마나 일치하는가**입니다.
빈 칸을 ATA 인자로 채우려면 그 인자가 코호트 간에 한목소리를 내야
믿고 곱할 수 있기 때문입니다. 인자가 코호트마다 들쭉날쭉하면 값이
1.5라 해도 그 1.5를 미래로 이어 붙이기 어렵습니다.

`ata()`는 인자(`f`)와 함께 그 인자가 코호트 간에 얼마나 안정적인지를
두 지표로 보고합니다.

- **CV**(변동계수, `cv`) — 같은 경과에서 코호트마다 인자가 얼마나
  흩어져 있는가. 작을수록 코호트들이 한목소리를 냅니다.
- **RSE**(상대표준오차, `rse`) — 합쳐 구한 인자 자체가 얼마나 정밀하게
  추정되었는가.

```python
import polars as pl
import lossratio as lr

df = lr.load_experience().filter(pl.col("coverage") == "SUR")
tri = lr.Triangle(df, groups="coverage", grain="Q")

tri.link(target="loss").ata().df.select(["dev", "f", "cv", "rse", "n_cohorts"]).head(4)
#> shape: (4, 5)
#> ┌─────┬──────────┬──────────┬──────────┬───────────┐
#> │ dev ┆ f        ┆ cv       ┆ rse      ┆ n_cohorts │
#> │ --- ┆ ---      ┆ ---      ┆ ---      ┆ ---       │
#> │ i64 ┆ f64      ┆ f64      ┆ f64      ┆ i64       │
#> ╞═════╪══════════╪══════════╪══════════╪═══════════╡
#> │ 1   ┆ 3.498967 ┆ 0.192339 ┆ 0.052384 ┆ 11        │
#> │ 2   ┆ 1.72324  ┆ 0.04679  ┆ 0.013441 ┆ 10        │
#> │ 3   ┆ 1.415805 ┆ 0.020162 ┆ 0.006801 ┆ 9         │
#> │ 4   ┆ 1.292501 ┆ 0.00745  ┆ 0.002987 ┆ 8         │
#> └─────┴──────────┴──────────┴──────────┴───────────┘
```

여기서 핵심을 놓치지 마십시오. 경과 1의 인자는 **3.5**로 손해가 한
분기에 3.5배 자라는, 결코 다 자라지 않은 구간입니다. 그런데 CV가 0.19로
커서, 코호트마다 그 3.5배가 들쭉날쭉합니다. 경과 2로 넘어가면 인자는
여전히 1.7로 손해가 한참 더 자라지만 CV는 0.05로 뚝 떨어집니다 — **자라는
크기는 아직 크지만, 자라는 패턴은 이미 코호트 간에 거의 일치**합니다.
즉 ATA 인자의 신뢰도는 손해의 *크기*가 아니라 코호트 간 *일치*가
가릅니다.

이 일치는 인자의 크기와 분리해서 봐야 합니다 — 아래에서 위(초기 경과)는
코호트마다 인자가 흩어져 CV가 크고(못 믿음), 아래(쌓인 경과)는 값이
커도 코호트 간 일치해 CV가 작습니다(믿을 만함). `o` = 코호트별 개별
인자:

```text
  immature dev: factors scattered across cohorts (high CV)
   f |  o       o
     |      o        o
     |   o      o
     +------------------- cohort
  stable dev: factors agree across cohorts (low CV)
   f |     oooooo
     |     oooooo
     +------------------- cohort
```

`link.plot(model="ata", kind="cv")`에 `show_factor_stability=True`를 주면,
인자의 CV·RSE가 문턱(`max_cv=0.15`, `max_rse=0.05`) 아래로 내려가 안정되는
경과 구간을 음영으로 표시해 줍니다.

```{eval-rst}
.. plot::
   :context:
   :nofigs:
   :include-source: false

   import polars as pl
   import lossratio as lr

   df = lr.load_experience().filter(pl.col("coverage") == "SUR")
   tri = lr.Triangle(df, groups="coverage", grain="Q")
   link = tri.link(target="loss", exposure="premium")

.. plot::
   :context: close-figs
   :caption: SUR 담보의 경과별 ATA 인자 CV(분기 단위). 음영은 인자가 코호트 간에 안정되는(CV·RSE 문턱 아래로 내려간) 구간이다.

   link.plot(model="ata", kind="cv", show_factor_stability=True)
```

```{admonition} 이 음영은 "손해 끝"이 아니다
:class: important

이 음영을 "손해가 다 났다"로 읽으면 안 됩니다. 음영이 시작되는 경과의
인자도 여전히 1보다 한참 큽니다 — 손해는 계속 자랍니다. 음영이 뜻하는
바는 **그 진전 패턴이 코호트 간에 일관되어 곱셈으로 미래를 이을 수 있게
되었다**는 것뿐입니다. 음영 이전(인자가 코호트마다 들쭉날쭉한 구간)에는
노출에 닻을 내린 3장의 ED가 안전하고, 음영 이후(패턴이 안정된 구간)에는
chain ladder가 손해의 곱셈식 성장을 직접 따라갑니다. 이 둘을 *어느
경과에서* 갈아탈지는 4장의 `lr.SwitchPoint`가 백테스트로 정합니다 —
이 CV·RSE 음영은 그 *판단의 재료*이지 전환점 자체가 아닙니다.
```

문턱을 더 엄격하게(작게) 잡으면 안정 구간은 그만큼 뒤로 밀립니다.
"얼마나 일관되어야 믿겠는가"를 분석자가 정하는 것입니다 — `max_cv` /
`max_rse` 인자로 조절합니다.

## 2.6 인자를 곱하면 — 누적 ATA 인자 곱

링크 하나의 인자가 "한 경과를 지날 때의 배율"이라면, 어떤 경과에서
끝(ultimate)까지의 인자를 **모두 곱한 값**은 "지금 손해가 최종에는 몇
배가 되는가"를 말해 줍니다. 이 누적 곱을 **누적 ATA 인자 곱**(LDF, Loss
Development Factor)이라 부릅니다.

예를 들어 앞 예제에서 경과 1의 누적 곱은 1.5 x 1.2 = 1.8입니다. 경과 1의
누적 손해에 1.8을 곱하면 그 코호트의 최종 손해 추정치가 됩니다. 빈 칸을
채우는 예측이 바로 이 곱셈을 이어 붙이는 일이며, 4장에서 직접 해 봅니다.
